## 1. 개요
RAG 파이프라인 성능 최적화를 위한 Phase 2 실험 결과를 바탕으로, 최적의 청킹 파라미터를 반영하고 실험 데이터 오염 방지를 위한 인프라 개선을 진행했습니다.

## 2. 주요 변경 사항

### 청킹 전략 최적화 (text_splitter.py)
- chunk_size: 500 → 800
- chunk_overlap: 50 → 100
- 근거: 특허 초록 분석 결과, 800자 상향 시 문맥 유실 위험이 감소함을 RAGAS로 확인

### 통계 API 네임스페이스 격리 (stats.py)
- 대시보드 통계 조회 시 namespace="" (운영 데이터)만 필터링하도록 수정
- 실험용 데이터(chunk_*)가 운영 지표에 영향을 주지 않도록 보장

### 파이프라인 유연성 확보 (rag_pipeline.py, vectorstore.py)
- 모든 검색 및 저장 로직에 namespace 파라미터 지원 추가
- full_content 필드 추가로 평가 시 전체 문맥 활용 가능

## 3. 청킹 비교 실험 결과

| Chunk/Overlap | Faithfulness | Precision | Recall |
|:---|:---|:---|:---|
| 500/50 | 0.7328 | 0.9401 | 0.9875 |
| **800/100** | **0.8089** | **0.9629** | **1.0000** |
| 1000/200 | 0.7155 | 0.9480 | 1.0000 |

> 500자는 문맥 단절로 정확도 하락, 1000자는 노이즈 증가로 Faithfulness 저하. **800자가 특허 도메인의 sweet spot.**

## 4. 영향 범위
- 신규 문서 인제스션 시 800자 단위로 분할됩니다
- 대시보드 통계 수치가 실험 데이터 제외 후 정상화됩니다
