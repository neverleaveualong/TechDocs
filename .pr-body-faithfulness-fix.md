## 1. 개요
RAG 답변 품질 핵심 지표(Faithfulness, Answer Relevancy) 저하 원인을 코드 레벨에서 해결. 프롬프트-컨텍스트 불일치로 인한 할루시네이션 구조적 제거 및 RAGAS 평가 정확도 개선.

## 2. 문제 분석

기존 RAGAS 평가 결과 (gt15 기준):

| 메트릭 | Vector Only | Hybrid |
|:---|:---|:---|
| Context Recall | 0.93 | 0.84 |
| **Faithfulness** | **0.28** | **0.36** |
| **Answer Relevancy** | **0.22** | **0.24** |

**근본 원인 3가지:**
1. 프롬프트는 출원번호/발명명칭/출원인을 "반드시 표시"하라고 요구하지만, context엔 본문(page_content)만 전달 → LLM이 메타데이터를 지어냄 (할루시네이션 강제 구조)
2. RAGAS 평가 시 `relevance_text[:200]`만 전달하여 Faithfulness 과소평가
3. 9개 프롬프트 규칙이 답변을 경직시켜 Answer Relevancy 저하

## 3. 주요 변경 사항

### ① Context에 메타데이터 포함 (`rag_pipeline.py`)
- `_build_context()` 메서드 추가: 각 chunk 앞에 [특허 정보] 헤더(출원번호, 발명명칭, 출원인, 출원일, 등록상태) 삽입
- LLM이 요구된 정보를 문서에서 직접 참조 가능 → 할루시네이션 제거
- 기대 효과: **Faithfulness 대폭 상승**

### ② 프롬프트 경량화 (`prompts.py`)
- 9개 규칙 → 5개로 축소
- 마크다운 금지, 영어 금지, 이모지 금지, 비교 구분 설명 등 불필요/모순적 규칙 제거
- 메타데이터 인용 규칙 추가 ("제공되지 않은 항목은 임의로 작성하지 마세요")
- 기대 효과: **Answer Relevancy 상승**, 답변 자연스러움 개선

### ③ RAGAS 평가 정확도 수정 (`evaluator.py`)
- `relevance_text[:200]` → `full_content` + 메타데이터 헤더로 변경
- RAGAS가 실제 LLM과 동일한 context를 평가에 사용
- 기대 효과: **측정 정확도 향상**, Faithfulness 과소평가 해소

## 4. 영향 범위
- `rag_pipeline.py`: context 조립 방식 변경 (모든 검색 API 응답에 적용)
- `prompts.py`: 답변 포맷 변화 (마크다운 허용, 규칙 축소)
- `evaluator.py`: 평가 결과 수치 변화 예상 (이전 결과와 직비교 시 주의)
