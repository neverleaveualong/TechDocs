## 1. 개요
Phase 3: Hybrid Search(BM25 + Vector + RRF) 도입으로 검색 품질 대폭 향상

## 2. 실험 결과 (RAGAS 10문항, 4개 메트릭)

| 실험 | Faithfulness | Precision | Recall |
|:---|:---|:---|:---|
| Vector Only (기존) | 0.7194 | 0.7350 | 0.9000 |
| **Hybrid (BM25+Vector+RRF)** | **0.7353** | **0.9589** | **1.0000** |
| Hybrid + Reranker | 0.4834 | 0.4589 | 0.4000 |

**Context Precision +30%, Context Recall +11% 향상**

Reranker(ms-marco-MiniLM)는 한국어에 부적합해 제외. **BM25+Vector+RRF 조합을 기본 검색으로 적용.**

## 3. 주요 변경 사항

### Hybrid Search 기본 적용
- `rag_pipeline.py`: `use_hybrid=True`로 기본값 변경
- BM25(키워드) + Vector(의미) + RRF 순위 병합

### 신규 모듈
- `hybrid_search.py`: BM25 + Vector + RRF 병합 로직, 한국어 토큰화
- `reranker.py`: Flashrank 기반 2차 정렬 (선택적, 한국어 개선 시 활용 가능)
- `run_hybrid_compare.py`: Phase 3 성능 비교 실험 스크립트

### API 확장
- POST /api/search에 `use_hybrid`, `use_reranker` 필드 추가

## 4. 아키텍처

```
Query -> Vector Search --|
                         +-> RRF 병합 -> top_k -> LLM
Query -> BM25 Search  ---|
```
